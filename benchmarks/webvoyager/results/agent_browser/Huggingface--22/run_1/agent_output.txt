ANSWER: Per the HF Transformers doc "How to convert a 🤗 Transformers model to TensorFlow?": **To add a TF architecture (7 steps):** 1) Select the model to convert; 2) Prepare the transformers dev environment (fork & clone the repo, `pip install -e ".[dev]"`); 3) (Optional) Understand the theory and the existing PyTorch implementation; 4) Implement the model architecture (create a `modeling_tf_*.py` file, port PyTorch ops to their TF equivalents); 5) Implement the model tests; 6) Submit the pull request; 7) (Optional) Build demos and share. Verify the port with `TFBrandNewBert.from_pretrained(model_repo, from_pt=True)` (outputs must match PyTorch within ~1e-5). **To convert existing PyTorch weights to TF weights and push to the Hub:** 1) Log in via `huggingface-cli login`; 2) Run `transformers-cli pt-to-tf --model-name foo/bar`; 3) Tag/notify HF team members to merge the resulting PR. Additionally, any TF model class can load PyTorch weights directly with `from_pretrained(..., from_pt=True)`.
